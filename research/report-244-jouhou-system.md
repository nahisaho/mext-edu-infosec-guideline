# 調査レポート：地方自治法 第244条の5・第244条の6 新設および地方自治法施行規則 第16条の3 新設

> [!NOTE]
> 本レポートは一次情報（総務省通知 PDF）を基にした事実整理と論点抽出であり、法解釈の最終見解ではない。条文の逐語表現・施行日は必ず e-Gov 法令検索および官報の原本で最終確認すること。
>
> 参考文献は [`references.md`](../references.md) を参照。

## 1. エグゼクティブサマリ

- **令和6年法律第65号**（地方自治法の一部を改正する法律、令和6年6月26日公布）により、地方自治法に **第244条の5**（情報システムの適正な利用等）と **第244条の6**（サイバーセキュリティ方針の策定・公表・総務大臣の指針）が新設された。
- **第244条の5**（第1項・第2項）は **令和6年9月26日**（公布日から3月経過日）に、**第244条の6**（第1〜3項）は **令和8年（2026年）4月1日** に施行される。
- **令和8年総務省令第80号**（地方自治法施行規則の一部を改正する省令、令和8年6月26日公布）により、施行規則に **第16条の3** が新設され、第244条の5第2項の「必要な措置」の内容として **8 分野の対策**（体制・情報資産管理・物理・人的・技術・運用・業務委託／外部サービス・評価見直し）が具体化された。**令和9年（2027年）7月1日施行**。
- 施行規則には、**技術的対策（第16条の3第1項第5号）** と **業務委託・クラウド利用（同第7号）** について **経過措置**（施行時点の既存システム・既存業務委託は「なお従前の例による」）が置かれた。
- 本改正は地方公共団体を対象とするが、教育情報セキュリティ（学校・教育委員会が扱う情報システム）は **設置者たる地方公共団体（教育委員会）の情報システム** として直接影響を受ける。したがって文部科学省「教育情報セキュリティポリシーに関するガイドライン」は、**地方自治法・施行規則の要請と整合する形で改訂する必要がある**。

## 3. 改正内容の詳細

### 3.1 地方自治法 第244条の5（情報システムの適正な利用等）

改正法「第一 二 情報システムの適正な利用等」に対応。

- **第1項**：普通地方公共団体は、事務の種類及び内容に応じ、第2条第14項・第15項（住民福祉の増進・最少の経費で最大の効果）の趣旨達成のため必要と認めるときは、**情報システムを有効に利用**するとともに、他の普通地方公共団体又は国と協力して **情報システムの利用の最適化**（広域・全国規模での共同利用、機能標準化等が想定される）を図るよう努めなければならない。（努力義務）
- **第2項**：普通地方公共団体は、情報システムの適正な利用を図るため **必要な措置** を講じなければならない。（義務。具体的内容は総務省令に委任 → 後述の施行規則第16条の3で規定）

### 3.2 地方自治法 第244条の6（サイバーセキュリティ方針の策定等）

- **第1項**：普通地方公共団体の **議会及び長その他の執行機関** は、サイバーセキュリティを確保するための **方針を定め**、及びこれに基づき必要な措置を講じなければならない。
  - 議会・長・その他の執行機関ごとに定めるのが原則だが、必要となる対策が概ね同様であって別個策定が非効率な場合、**複数機関による共同策定**が可能（通知の解説）。
- **第2項**：方針を定め、又は変更したときは、**遅滞なく公表** しなければならない。
- **第3項**：総務大臣は、方針の策定・変更について **指針を示す** とともに、**必要な助言** を行う。
  - 通知は、今後総務大臣が定める指針において「各地方公共団体のセキュリティ対策における基本的な考え方を示す予定」と明記。**指針の内容を踏まえて方針を策定すること** が期待されている。
- **準用**：一部事務組合・広域連合について地方自治法第292条により準用（通知）。地方独立行政法人については **地方独立行政法人法第24条の2** により、**第244条の5第2項及び第244条の6第1項〜第3項** が準用される（令和6年通知「第五 二」に「第一の二の2から5までの情報システムの適正な利用に関する規定」が地方独立行政法人について準用される旨明記）。

### 3.4 地方自治法施行規則 第16条の3（令和8年総務省令第80号による新設）

第244条の5第2項に基づく「必要な措置」を8分野で具体化。

| 号 | 措置 |
| --- | --- |
| 第1号 | 組織体制の整備 |
| 第2号 | 適切な情報資産の分類及び管理の実施 |
| 第3号 | 物理的なサイバーセキュリティに関する対策の適切な実施 |
| 第4号 | 人的なサイバーセキュリティに関する対策の適切な実施 |
| 第5号 | 技術的なサイバーセキュリティに関する対策の適切な実施（情報システム等の開発・導入・保守の適切な実施、サイバーセキュリティに関する情報の収集を含む） |
| 第6号 | サイバーセキュリティに関する対策の適切な運用の実施 |
| 第7号 | 適切な業務委託（情報システム等に関するものを含む）の実施、及び外部サービス（クラウドサービス）の適切な利用 |
| 第8号 | サイバーセキュリティに関する対策の適切な評価及び見直しの実施 |

**適用除外**：国又は他の地方公共団体の重要な情報又は重要な情報システムに影響を及ぼす可能性があるネットワークに電気通信回線で直接又は間接に接続されておらず、かつ **個人情報を多量に保有していない団体** 及び **公安委員会** は、上記措置の対象外。

**施行期日**：令和9年（2027年）7月1日。

**経過措置**：施行の際現に存する情報システム等及び現に実施されている業務委託については、**第5号（技術的対策）** 及び **第7号（業務委託・クラウド）** の規定にかかわらず、なお従前の例による。

### 3.5 別紙1：サプライチェーン・リスク対策

施行規則第16条の3第1項第5号（開発・導入・保守）及び第7号（業務委託・クラウド）の実施要領として、通知の別紙1が詳細を規定。要点：

- **対象情報資産**：ネットワーク（通信回線・ルータ等）、情報システム（サーバ、PC、モバイル端末、汎用機、複合機、OS、ソフトウェア、ドローン、ネットワークカメラ、クラウドサービス等）、施設・設備（コンピュータ室、配電盤、電源／通信ケーブル等）、電磁的記録媒体（内蔵／外付け／USB／DVD等）、取り扱う情報、関連文書（設計書、仕様書、マニュアル、ネットワーク構成図等）。
- **サプライチェーン・リスクの定義**：機器等の開発・供給事業者、設置・保守事業者、及びそれらのサプライヤー・（再）委託先について、**本社等の所在地の法的環境や外部主体の指示等** により、機器等の適切性が影響を受け、悪意ある機能や不正な変更が組み込まれる、又は情報が窃取・破壊される等のリスク。
  - 「本社等」には、当該事業者の総株主等の議決権の過半数を直接又は間接に保有する者の本社等を含む。
- **調達可否の判断**：リスクを受容／低減する措置が可能かを十分検討した上で決定。
- **クラウド利用**：データセンター所在国の法律により、**令状主義・透明性・不利益処分手続等の適切な手続によらずに** クラウド内情報が外国法執行機関の命令で強制開示されるリスクがあると判断される場合、当該クラウド利用を **行わない** ことを含む。
- **推奨評価制度**：情報資産の分類に応じ、原則として次のいずれかへの登録製品を調達。
  - **JC-STAR**（セキュリティ要件適合評価及びラベリング制度、IPA）
  - **ISMAP／ISMAP-LIU**（政府情報システムのためのセキュリティ評価制度）
  - **DMP**（デジタルマーケットプレイス、デジタル庁）
- **契約方式**：**総合評価一般競争入札**（地方自治法施行令第167条の10の2第1項・第2項）等、調達内容に応じた適切な方式を選定。
- **総合相談窓口**：総務省に設置予定。詳細は今後通知。

### 3.6 別紙2：脆弱性診断

- 総務省が令和8年度中に、全地方公共団体が利用可能な **地方版アタックサーフェスマネジメント（ASM）システム** を一括構築、効果実証後に本格運用を開始予定。
- 施行規則第16条の3第1項第5号の「サイバーセキュリティに関する情報の収集」の具体的実施手法の一つとして位置付けられる。

## 4. 文部科学省ガイドラインへの影響分析

### 4.1 現行ガイドラインの位置付け

- 文部科学省「教育情報セキュリティポリシーに関するガイドライン」は、**学校及び教育委員会** における情報セキュリティポリシー策定の指針。
- 教育委員会は地方公共団体の執行機関の一つであり、**第244条の6第1項が定める「その他の執行機関」に該当する** 可能性が高い（要確認：地方教育行政の組織及び運営に関する法律との関係整理が必要）。
- 第244条の6の方針策定・公表義務は執行機関ごとに課されるため、教育委員会は自らのサイバーセキュリティ方針を策定・公表する立場になり得る。
- 一方、第244条の5第2項に基づく施行規則第16条の3の8分野措置は、条文上の義務主体が **普通地方公共団体** であり、教育委員会所管の情報システム（校務系・学習系ネットワーク、GIGA 端末等）における実施分担は、当該地方公共団体の内部で別途整理する必要がある。この分担のあり方はガイドライン改訂の主要論点となる。

### 4.2 想定される整合性論点

以下は改訂検討で優先的に扱うべき論点。**各論点は要検証**。

1. **教育委員会独自の方針策定か、首長部局と統合か**
   - 第244条の6第1項は執行機関ごと策定を原則としつつ共同策定を許容。教育委員会が独自方針を持つべきか、地方公共団体全体の方針の下位に位置付けるかは、教育情報の特性（児童生徒の個人情報、成績等の要保護度）を踏まえた文科省の指針が必要。
2. **総務大臣の「指針」と文科省ガイドラインの関係**
   - 第244条の6第3項の総務大臣指針が「基本的な考え方」を示す想定であるため、文科省ガイドラインは **総務大臣指針と整合し、これを参照する構造** に再編する必要がある可能性（総務大臣指針・総務省通知はいずれも第245条の4第1項に基づく技術的助言であり、上下関係ではなく整合性の観点で整理する）。
3. **8分野措置の教育現場向け読替え**
   - 施行規則第16条の3各号は汎用的な規定。学校・教育委員会の実務（GIGA スクール端末、学習系ネットワーク、校務系ネットワーク、統合型校務支援システム、クラウド教材・学習eポータル等）に即した具体化が必要。
4. **サプライチェーン・リスク対策（別紙1）と教育調達**
   - JC-STAR／ISMAP／DMP 登録製品の優先調達方針が、GIGA 端末調達・教育クラウド調達に与える影響。特に **海外クラウドサービス（LMS、生成AI教材、ドリル教材等）の利用可否判断基準** の整理が必要。
   - 学校設置者（市町村）レベルでの調達判断能力を補うため、文科省による共通評価・推奨リストの提供が実効性を高める可能性。
5. **経過措置の適用範囲**
   - 令和9年7月1日時点で既に運用中のシステム・業務委託は第5号・第7号について「なお従前の例による」。既存の GIGA 端末・LMS・校務支援システムの多くが対象となる可能性があり、**更改タイミングとの整合** をガイドラインで示す必要。
6. **公表義務（第244条の6第2項）と学校現場**
   - 方針の公表が児童生徒・保護者・地域住民に対する説明責任を強化する一方、詳細な技術情報の公開は攻撃者への情報提供リスクを伴う。**公表範囲・粒度に関する指針** が必要。
7. **脆弱性診断（別紙2、地方版 ASM）の学校 IT 資産への適用**
   - 学校が保有する Web サイト、学習系サーバ、校務系サーバ、公開機器（ネットワークカメラ等）の診断対象化。診断結果への対応体制（教育委員会 CSIRT の位置付け等）の整備。
8. **一部事務組合・広域連合・地方独立行政法人の準用**
   - 教育分野で該当する主体（教育センター、教育に係る事務組合、学校法人化された公立大学法人等）への適用整理。
9. **公安委員会型の適用除外規定の可能性**
   - 施行規則の適用除外条件（重要ネットワーク非接続かつ個人情報を多量に保有しない）は、教育委員会・学校の実態（多量の児童生徒個人情報を保有）から見て **ほぼ該当しない** と想定される。ガイドラインで明示的に「原則として全教育委員会・学校が対象」と示すことが望ましい。

## 6. 改訂プロセスに関する提案

1. **総務大臣指針の公表を待つ**：第244条の6第3項の指針が示された時点で、文科省ガイドラインを整合させることが最も効率的。指針の公表時期を総務省自治行政局住民制度課サイバーセキュリティ対策室（連絡先：lg-security@soumu.go.jp）に照会することが望ましい。総務大臣指針と文科省ガイドラインはいずれも技術的助言としての性質を持つため、法的上下関係ではなく **整合性** の観点で位置付ける。
2. **段階的改訂**：令和8年4月1日（第244条の6施行）と令和9年7月1日（施行規則施行）の2段階で改訂版を出す構成。
3. **設置者別ガイド**：都道府県教育委員会・市町村教育委員会・国立学校・私立学校で適用構造が異なるため、対象別に読替表を用意。
4. **移行支援ツール**：既存ポリシー ↔ 新ガイドラインの対応表、教育委員会向け方針ひな型、8分野チェックリスト、調達仕様書テンプレートの整備。
